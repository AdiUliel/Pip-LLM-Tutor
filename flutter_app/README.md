# Emotional Tutor — Parent App

Flutter + Firebase **parent companion** for an ESP32-based emotional tutor for kids (ages 7–11). This app does **no tutoring** — that lives on the physical device (mic + speaker + ILI9341 display + button). This app lets a parent configure the child, upload material, view reports + mood/accuracy trends, and monitor the device.

B.Sc IoT project, Technion — Group #02. See `User_Stories_Emotional_Tutor.pdf` (one folder up) for the original user-stories spec.

## Quick start

```bash
flutter pub get
flutter run        # local dev
flutter build web --release && firebase deploy --only hosting   # publish
```

The app always talks to the real Firebase project (`llm-tutor-d721e`). Auth uses Firebase Email/Password; parent + child + material + session data live in Firestore; uploads go to Storage. Make sure **Authentication → Sign-in method → Email/Password** is enabled in the [Firebase console](https://console.firebase.google.com/project/llm-tutor-d721e/authentication/providers).

Hosted URL: https://llm-tutor-d721e.web.app

## Reconfigure the Firebase project

If you need to point the app at a different Firebase project:

1. Install the Firebase CLI and the FlutterFire CLI:
   ```bash
   curl -sL https://firebase.tools | bash
   dart pub global activate flutterfire_cli
   ```
2. From the project root, run:
   ```bash
   flutterfire configure
   ```
   Pick the project; this regenerates `lib/firebase_options.dart`.

## Firestore data model (shared with the ESP32 device)

```
parents/{parentId}                     # WRITTEN BY APP
  name, email, createdAt

children/{childId}                     # WRITTEN BY APP
  parentId, name, age,
  subjectsEnabled: ["math","english"],
  topicFocus: { math: "multiplication", english: "spelling" },
  level: { math: 1, english: 1 },
  settings: { sessionMinutes, breakEveryMinutes, dailyLimitMinutes },
  deviceId, createdAt

materials/{materialId}                 # WRITTEN BY APP
  childId, subject, title,
  items: [ { question, answer } ],     # typed Q/A pairs
  fileUrl,                             # uploaded homework blob (Storage)
  uploadedAt

deviceState/{deviceId}                 # device writes; app reads
  online, lastHeartbeat,
  status: "idle"|"asking"|"listening"|"feedback"|"break"|"error",
  activeSubject, currentQuestion,
  command: "start" | "stop" | "none"   # OPTIONAL — app may write for remote control

sessions/{sessionId}                   # device writes; app reads
  childId, subject, startedAt, endedAt,
  questionsAsked, correctCount, wrongCount, accuracy,
  starsEarned, longestStreak, moodSummary
  questions/ (subcollection)           # device writes
    { prompt, expectedAnswer, childAnswerTranscript,
      correct, mood, difficulty, askedAt }
```

The child starts/ends a session with the **device's physical button**. The device emits its `status` + a periodic `lastHeartbeat`; the parent app derives "online" from the heartbeat (timeout in `constants.dart`).
