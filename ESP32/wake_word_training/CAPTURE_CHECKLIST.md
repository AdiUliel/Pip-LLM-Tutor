# צ'קליסט: הקלטות אמיתיות + אימון-מחדש של "hey pip"

המודל הנוכחי אומן רק על קולות סינתטיים — הקלטות אמיתיות דרך המיקרופון של המכשיר
הן מה שסוגר את הפער (גם הפספוסים וגם היריות על רעש). כל התהליך: ~15 דק' הקלטות
+ ~30 דק' אימון במחשב.

## שלב 1 — הקלטה על המכשיר (~15 דק')

1. להכניס כרטיס SD למכשיר.
2. ב-`homework_assistant.ino` לקבוע `WW_CAPTURE_MODE 1` ולצרוב.
3. לפתוח Serial Monitor ‏(115200). המכשיר מקליט לפי מקשים בסריאל:
   - **`p`** (או לחיצת הכפתור) — הקלטת "hey pip" אחת (1.8 שנ').
   - **`n`** — משפט קצר שהוא *לא* מילת ההשכמה (1.5 שנ').
   - **`c`** — 20 שניות של רעש הרקע של החדר.
   - **`l`** — הצגת ספירה + מקום פנוי.

   מה להקליט (המטרות מודפסות גם בסריאל):

   | מה | כמה | טיפים |
   |----|-----|-------|
   | "hey pip" (`p`) | **100–150** | הילד/ה עצמם + בני משפחה; מרחקים שונים (0.5–3 מ'), טונים שונים (רגיל/מהיר/עייף/צועק); גם כמה עם טלוויזיה ברקע |
   | משפטים אחרים (`n`) | **~30** | דיבור עברית רגיל, שמות אחים, מילים דומות ("hey", "pip", "פיצה") |
   | אווירת חדר (`c`) | **~8–10 דק'** (25–30 הקלטות) | החדר האמיתי: שקט, טלוויזיה, מאוורר/מזגן, אחים משחקים — במיוחד הרעשים שגרמו ליריות-שווא |

   ⚠️ אם הסריאל מזהיר `clipped` — להתרחק מהמיקרופון; `too quiet` — להתקרב.
4. לכבות, להוציא את ה-SD, להעתיק את תיקיית **`/ww`** למחשב (למשל `~/ww`).
5. להחזיר `WW_CAPTURE_MODE 0` בצריבה הבאה.

## שלב 2 — אימון במחשב (~30 דק', לפי RETRAIN_RUNBOOK.md)

```bash
cd ESP32/wake_word_training
python3 -m venv ~/wwenv && source ~/wwenv/bin/activate
pip install tensorflow-cpu numpy scipy soundfile edge-tts espeakng_loader py-espeak-ng
# ffmpeg נדרש ל-edge-tts (ב-mac: brew install ffmpeg; ב-ubuntu: apt install ffmpeg espeak-ng)

rm -rf /tmp/ww && mkdir -p /tmp/ww/out          # דאטהסט נקי — בלי שאריות סינתטיות ישנות

python3 build_dataset.py --ww ~/ww               # אין אינטרנט? להוסיף --no-tts
python3 train.py                                 # יעד: val acc ≥ 0.97, ~0 יריות-שווא ב-0.7–0.8
python3 export_c.py                              # כותב hey_pip_model.h חדש
python3 parity.py                                # חייב להדפיס PASS לפני צריבה!
```

`train.py` מדפיס טבלת סף (precision/recall ב-0.5/0.7/0.8/0.9) — לרשום איזה סף נותן
recall גבוה עם ~0 יריות-שווא; זה ה-`WAKE_WORD_THRESHOLD` המומלץ החדש.

## שלב 3 — פריסה

1. `cp /tmp/ww/out/hey_pip_model.h ../homework_assistant/hey_pip_model.h`
2. לצרוב עם `USE_WAKE_WORD 1`.
3. לכייל סף לפי המתכון ב-`Documentation/WAKE_WORD.md` (‏WW_TEST_MODE, ‏3 דקות),
   ולהריץ את צ'קליסט הבדיקות T1–T7 שם.

עדיין לא מספיק טוב? סדר העדיפויות ב-RETRAIN_RUNBOOK.md §9 — קודם עוד הקלטות
אמיתיות (במיוחד הרעשים הספציפיים שיורים), רק בסוף כוונון ספים.
